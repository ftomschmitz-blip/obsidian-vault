Säule 1: Der Git-Sync (Die Brücke)

Der Git-Sync ist das unumgängliche Fundament deiner intelligenten Agentenlandschaft. Dadurch arbeiten sowohl du lokal als auch dein Hermes Agent auf dem VPS an ein und demselben Vault, ohne das Daten verloren gehen.

  1. 1.1 Privates GitHub Repository erstellen
    Erstelle auf GitHub ein neues Repository namens obsidian-vault. Stelle es zwingend auf Private. Wichtig: Initialisiere es nicht mit einer README oder .gitignore (lasse es komplett leer!).
  2. 1.2 GitHub Personal Access Token (PAT) generieren
    Navigiere auf GitHub zu Settings -> Developer Settings -> Personal access tokens -> Tokens (classic). Erstelle ein neues Token mit dem Scope repo und kopiere das Token sofort (Format: ghp_...).
  3. 1.3 Lokal git konfigurieren und pushen (Mac)
    Öffne das Terminal auf deinem Mac und navigiere in deinen lokalen Obsidian-Ordner (Pfade mit Leerzeichen in Anführungszeichen setzen!): 
    cd "/Users/tschmitz/ObsidianVaults/Second Brain Tom"
git init
git remote add origin https://github.com/ftomschmitz-blip/obsidian-vault.git
git add .
git commit -m "Initialer Backup"
git branch -M main
git push -u origin main
    (Gib bei Passwortabfragen dein kopiertes ghp_-Token ein).
  4. 1.4 Obsidian Git Plugin konfigurieren (Lokal)
    Installiere in Obsidian das Community-Plugin Obsidian Git. Scrolle in den Plugin-Einstellungen ganz nach oben zur Sektion "Automatic":
    • Auto commit interval (minutes): Setze dies auf 15
    • Auto push interval (minutes): Setze dies auf 15
  5. 1.5 Automatischer Pull auf dem VPS (Cron-Job)
    Auf dem VPS ist dieser Schritt durch deinen aktiven Cron-Job 141e7cc673f9 und das Skript obsidian-sync.sh bereits perfekt gelöst. Er holt alle 30 Minuten deine Änderungen ab.

Säule 2: Lokaler Claude-Obsidian-MCP

Damit dein lokaler Claude Desktop (oder Tools wie Cursor) in Echtzeit deine lokalen Notizen lesen und verknüpfen kann, binden wir Obsidian direkt lokal ein.

  1. 2.1 Local REST API Plugin in Obsidian installieren
    Suche in Obsidian nach dem Community-Plugin Local REST API, installiere und aktiviere es.
    • Aktiviere HTTPS in den Plugin-Einstellungen.
    • Generiere ein API-Token und kopiere es.
  2. 2.2 Obsidian MCP-Server konfigurieren
    Öffne die Konfigurationsdatei deines lokalen Claude Desktops unter Mac OS: ~/Library/Application Support/Claude/claude_desktop_config.json.
    Trage den Obsidian-Server von Markus Pfundstein ein: 
    {
  "mcpServers": {
    "obsidian": {
      "command": "docker",
      "args": [
        "run",
        "-i",
        "--rm",
        "-e",
        "OBSIDIAN_API_KEY=dein_api_key_hier",
        "-e",
        "OBSIDIAN_PORT=27124",
        "mcp/server/obsidian"
      ]
    }
  }
}
💡 Profi-Tipp: Falls du kein Docker installiert hast, kannst du das Tool auch über npx / Node.js direkt einbinden.

Säule 3: Remote VPS-Hermes-MCP (SSH-Bridge)

Dies verbindet das lokale Claude-Gehirn mit der VPS-Muskelkraft. Dein lokaler Claude steuert mich direkt auf dem VPS per SSH.

  1. 3.1 SSH-Key-Authentifizierung einrichten
    Damit Claude sich nahtlos verbinden kann, darf kein Passwort abgefragt werden. Richte einen passwortlosen SSH-Key ein: 
    ssh-copy-id dein_vps_user@deine_vps_ip
  2. 3.2 SSH-MCP in Claude Desktop eintragen
    Erweitere deine claude_desktop_config.json um meinen MCP-Dienst auf dem Hostinger VPS: 
    {
  "mcpServers": {
    "hermes-vps": {
      "command": "ssh",
      "args": [
        "-t",
        "dein_vps_user@deine_vps_ip",
        "/opt/hermes/.venv/bin/hermes mcp serve"
      ]
    }
  }
}

Best Practices & Absicherung

Merge-Konflikte meiden

Erstelle im Root deines Vaults eine .gitignore-Datei mit folgenden Zeilen, um temporäre Layout-Änderungen von Obsidian vom Synchronisieren auszuschließen:

.obsidian/workspace.json
.obsidian/workspace-mobile.json
.obsidian/plugins/obsidian-git/

Asynchrone Agent-Inbox

Erstelle eine Notiz Inbox/Agent_Inbox.md. Schreibe dort Aufgaben im Checkbox-Format auf (z.B. - [ ] Aufgabe: ...). Ein VPS-Cronjob kann diese im Hintergrund abarbeiten, Ergebnisse schreiben und abhaken, während du offline bist.

Sichere Befehle

Setze auf dem VPS die Sicherheitsstufe in der Config auf approvals.mode: smart. So führt Hermes im Chat keine destruktiven Befehle unbestätigt aus, während im lokalen MCP-Modus über SSH freie Hand herrscht.
